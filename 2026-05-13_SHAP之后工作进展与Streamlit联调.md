## 一、推理封装（`inference_utils.py`）

### 1.1 函数清单

| 函数 | 功能 | 验证状态 |
|:-----|:------|:--------:|
| `load_dual_model()` | 加载 v2.0 Attention-LSTM 权重 | ✅ |
| `predict()` | 单次推理 → thrust(N), mfr(mg/s), Isp(s) | ✅ |
| `predict_with_uncertainty()` | MC Dropout ×N 次 → 均值 ± 标准差 | ✅ |
| `compute_residuals()` | 残差计算 + 阈值异常检测 | ✅ |
| `generate_health_report()` | 基于残差的健康诊断报告（重构版） | ✅ |

### 1.2 关键修复

| 修复 | 问题 | 解决 |
|:-----|:-----|:------|
| **Isp 单位换算 bug** | `1e-3`(g/s) 应为 `1e-6`(kg/s)，导致 Isp 偏小 1000 倍 | 改为 `1e-6`，Isp 从 0.2s → 正确值 ~160s |
| **健康评分公式** | 基于绝对阈值（thrust>1N, Isp>150s），低压工况全 critical | 改为基于**预测残差 RMS**，以模型 RMSE=0.0832N 校准 |
| **评分规则透明化** | 无说明，评委看不懂 | 报告底部嵌入评分规则说明 + 权重公式 |

### 2.3 数据加载双模式

| 模式 | 条件 | 流程 |
|:-----|:------|:------|
| **直接推理** | CSV 文件名在 metadata.csv 中有匹配 | `load_sequence_window` → `predict_with_uncertainty`(MC Dropout) |
| **缓存回放** | 文件名不在 metadata 中 | 从 `predictions_dual.npy` 提取对应样本 |

### 2.4 自适应点火检测

从 CSV 中自动检测推进器点火帧（第一个 `ton>0`），跳过后启动瞬态段（80 帧缓冲），确保窗口落在稳定燃烧段。

### 2.5 修复汇总

| 问题 | 根因 | 修复 |
|:-----|:------|:------|
| 中文字体方框 | WSL matplotlib 无可信中文字体 | 安装 `fonts-noto-cjk` + `FontProperties` 硬编码 |
| Logo 不显示 | Streamlit 不支持本地路径 `<img src="assets/...">` | Base64 内嵌图像 |
| 推力全 0 | offset 未跳过点火前空闲段 | 自适应 ton>0 检测 + 跳 80 帧瞬态 |
| Isp=0.1s | mg/s→kg/s 换算用错 `1e-3` 而非 `1e-6` | 修正换算系数 |
| 健康评分全 0 | 评分公式对残差 >0.05N 直接判 0 分 | 以模型 RMSE 校准的多段评分 |
| 报告无评分规则 | 缺失 | 报告底部嵌入规则说明 + 综合权重 |

## 三、健康报告评分体系

### 3.1 三维度评分

| 维度 | 权重 | 评分依据 |
|:-----|:----:|:---------|
| 预测精度 | 0.4 | 残差 RMS / 模型 RMSE(0.0832N)，比值 ≤1.5→100分，≤3→80~100分，≤5→10~40分，>5→10分 |
| 异常检测 | 0.3 | 异常步占比，0%→100分，20%→0分，线性插值 |
| 模型置信度 | 0.3 | MC Dropout ×20 次变异系数 CV=std/mean，CV<3%→98分，<8%→85分，<15%→65分，<30%→40分，>30%→20分 |

> 综合 = 精度×0.4 + 异常×0.3 + 置信×0.3

### 3.2 状态等级

| 分数 | 等级 | 说明 |
|:----:|:----:|:------|
| ≥ 70 | 🟢 GOOD | 运行正常 |
| 40-69 | 🟡 WARNING | 需关注 |
| < 40 | 🔴 CRITICAL | 建议检修 |

## 四、已知问题

1. **图表 DPI** —— Streamlit 默认渲染降低 matplotlib DPI，已用 `render_fig()` 的 PNG buffer + 180dpi 缓解
2. **MC Dropout 延迟** —— 首次推理需 ~3-5 秒（20 次前向），已缓存 `@st.cache_resource`
3. **SSF 24bar 异常率偏高** —— 部分 CSV 文件瞬态特征导致残差大，需进一步分析
