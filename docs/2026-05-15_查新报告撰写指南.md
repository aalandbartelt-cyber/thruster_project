### 2.3 创新要点

直接列出 5 个创新点（见 §1.3），每点 2-3 句话展开：

```
1. 多源异质动态特征融合
   本项目首次在推进器健康监测中将6维连续工程统计量（ton均值/标准差、vl均值/标准差、
   test_pressure、窗口内趋势）与11维工况one-hot编码联合构成17维异质输入特征，覆盖
   催化床退化的三种独立物理机制——时间疲劳（cumulated_on_time）、化学侵蚀
   （cumulated_throughput）、热冲击（cumulated_pulses）。SHAP分析表明三大老化
   因子合计占推力预测归因的49%。

2. 多目标耦合预测架构
   设计 Attention-LSTM 双输出架构（256维隐层，4头注意力），联合预测推力与质量流量，
   损失函数采用加权MSE：Loss = 0.7 × MSE(thrust) + 0.3 × MSE(mfr)。在线推导
   比冲 Isp = F / (ṁ × g₀)，构建"推力-流量-比冲"三维健康指标体系。测试集推力RMSE
   达0.0832N，质量流量RMSE达49.1mg/s。

3. 基于真实标签的可量化异常检测
   利用数据集中270个真实逐点异常标签（覆盖31种异常类型），在残差检测基础上计算
   Precision / Recall / F1-Score / IoU 四项定量评估指标，替代传统经验阈值方法。

4. 个性化迁移学习应对制造公差
   采用冻结LSTM+Attention层、极小学习率（1e-4）微调输出层的策略，使全局预训练
   模型适配个体推进器的制造公差。12台推进器微调实验表明，平均推力RMSE改善4.6%。

5. 国产化算力适配验证
   同一Attention-LSTM模型在华为Ascend NPU（CANN 8.3.RC1）上成功完成57 epoch
   训练，收敛至best val_loss=0.000995，架构/数据/超参与CUDA版完全对齐。验证了
   推进器健康监测系统可在国产硬件上完整运行，不依赖CUDA生态。
```

## 三、关键注意事项

### 3.1 查新报告的核心逻辑

查新报告的本质是**证明"没有人做过和我们一模一样的事"**，而非证明"我们做的事没人碰过"。

- 如果检索到很接近的文献 → 承认其存在，但指出具体差异（特征/架构/应用对象/输出维度等）
- 如果检索结果很少 → 扩大检索词范围，至少要有 6-8 篇可以对比的文献
- **不要声称"完全没有相关研究"** —— 会被评委认为不会检索文献

### 3.2 与研究报告的关系

查新报告的 3 个查新点应对应研究报告"创新点"章节中的 5 个核心创新点，形成互文证据链：
- 查新点1 → 创新点1+2
- 查新点2 → 创新点1+6
- 查新点3 → 创新点5

### 3.3 盖章注意事项

- 查新报告需要**学院盖章**，与报名表的 E 栏是同一枚章（可同时送审）
- 签字要求：全体队员在"申报者签字"处签名
- 盖章时间：建议 5/16（周五）上午送学院，与报名表盖章一并办理

### 3.4 格式规范

- 使用 A4 纸打印，上下左右页边距不小于 2.5cm
- 正文字号：小四号宋体（12pt），标题用小三号黑体（15pt）
- 参考文献按 GB/T 7714-2015 格式著录
- 最终输出为 PDF，上传报名系统
