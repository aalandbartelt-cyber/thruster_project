### 4.3 创建 SSH 密钥对（安全登录用）

## 五、VS Code 连接华为云服务器（5/12 晚 ~ 5/13）

### 5.1 配置 SSH 连接

1. 打开 VS Code
2. 点击左下角蓝绿色 `><` 图标
3. 选择 **「Connect to Host...」**
4. 选择 **「Add New SSH Host...」**
5. 在弹出的框里输入（**把 IP 换成你的服务器公网IP，把密钥路径换成你的**）：

   ```
   ssh root@你的服务器公网IP -i C:\Users\你的用户名\.ssh\ascend-key.pem
   ```

   > 例如：`ssh root@123.123.123.123 -i C:\Users\mary\.ssh\ascend-key.pem`

6. 回车 → 选择第一个 SSH 配置文件（`C:\Users\你的用户名\.ssh\config`）
7. 提示「Host added!」→ 点击 **「Connect」**
8. 新开一个 VS Code 窗口 → 点 **「Connect」** → 第一次连接会问你信任这个主机，选「Continue」
9. 连接成功后，VS Code 左下角会显示 **`SSH: thruster-ascend`**（或你的服务器 IP）
10. 点击 VS Code 顶部菜单 **「Terminal → New Terminal」**，下方会出现一个终端

> **验证连接成功：** 终端里应该显示 `root@thruster-ascend:~#` 之类的提示符

### 5.2 验证环境是否正常

在 VS Code 终端里依次执行以下命令：

```bash
# 1️⃣ 查看 NPU 芯片信息
npu-smi info
```

**预期输出：** 显示昇腾芯片信息，CANN Version: 9.0.0

```
NPU ID  Name                Health  Power     Temp    Memory
0       昇腾xxx             OK      70.0W     45C     32GB/32GB
```

<details>
<summary>❌ 如果报错「npu-smi: command not found」</summary>
说明你选的镜像不对，不是 CANN 9.0 + PyTorch 的镜像。
回到 ECS 控制台 → 更多 → 重装系统 → 重新选镜像「人工智能平台 → CANN 9.0 + PyTorch 2.4.0」
</details>

<details>
<summary>❌ 如果显示无 NPU 设备</summary>
说明你选的 CPU 架构可能是 x86 而不是「鲲鹏计算」。需要删除服务器重新创建。
</details>

```bash
# 2️⃣ 验证 PyTorch 和 torch_npu
python3 -c "import torch; import torch_npu; print('PyTorch:', torch.__version__); print('NPU可用:', torch.npu.is_available())"
```

**预期输出：**
```
PyTorch: 2.4.0
NPU可用: True
```

<details>
<summary>❌ 如果报错「No module named torch_npu」</summary>
镜像没选对，重装系统选 CANN 9.0 + PyTorch 的镜像。
</details>

## 十四、常见问题大全

| 问题 | 原因 | 解决方法 |
|:-----|:-----|:---------|
| **VS Code 连不上** `ssh: connect to host ... port 22: Connection refused` | 服务器还没创建好或没开机 | 去华为云控制台确认服务器状态为「运行中」 |
| **VS Code 连不上** `Permission denied (publickey)` | 密钥不对 | 确认 -i 参数指向正确的 .pem 文件路径 |
| **`npu-smi: command not found`** | 镜像没选对 | 重装系统，选「CANN 9.0 + PyTorch 2.4.0」 |
| **`NPU 不可用: False`** | 架构不是鲲鹏 | 检查服务器 CPU 架构，需是「鲲鹏计算」 |
| **`import torch_npu` 报错** | 环境没装好 | 用 `pip3 list \| grep torch` 确认已安装 |
| **训练中途断连** | 网络超时 | 重新 VS Code 连接 → 终端运行 `screen -r` 或重跑 |
| **`out of memory`** | 显存爆了 | 把 `batch_size=64` 改成 `batch_size=32`（在 model_train_v2.py 里改） |
| **运行没反应** | 在等 GPU | 昇腾训练比 CUDA 慢是正常的，等着就行 |
