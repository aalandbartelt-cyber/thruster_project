# 优化方向：个体推进器微调（Per-Thruster Fine-Tuning）

> 编写日期：2026-05-13  
> 前置工作：v1→v2 全局模型已完成，v1 已有 `model_finetune.py` 可复用经验  
> 目标：解决"一个模型适配所有推进器导致误差较大"的问题

---

## 一、问题背景

当前 v2.0 模型使用全部 908 个正常文件（SN01-SN12）训练，服务所有推进器（SN13-SN24 测试集）。

**现有问题：**
- 全局模型 thrust RMSE = 0.0832N，但按 test_mode 分项后 ssf 高达 0.1390N
- 不同 SN 之间推力水平差可达 2 倍（SN13 mean=0.65N vs SN18 mean=0.97N），已在 v2.1 实验中发现
- 制造公差导致同一工况下不同推进器的响应特性不同

**差异化根源：**
- 推进器制造公差：催化床填充量、喷嘴尺寸、阀门响应时间差异
- 使用历史差异：cumulated_on_time / throughput / pulses 三个老化因子无法完全刻画个体差异

## 二、v1 微调经验回顾

v1 的 `model_finetune.py` 方案：

| 项目 | 值 |
|:-----|:----|
| 预训练模型 | v1 SimpleLSTM（3 维单输出） |
| 微调目标 | SN13（单台推进器） |
| 学习率 | **1e-4**（远低于初始训练的 1e-3） |
| 数据量 | 该 SN 的全部正常文件 |
| 冻结策略 | 全部权重参与微调（无冻结层） |
| 效果 | RMSE 降低 **27.97%** |

## 三、v2 微调方案设计

### 3.1 核心思路

```
全局模型（17维 Attention-LSTM）
        ↓ 保存 .pth
个体微调（按 SN 或按 test_mode）
        ↓
多个专用模型：
  ├── dual_output_lstm_v2_sn13.pth   (SN13 专用)
  ├── dual_output_lstm_v2_sn14.pth   (SN14 专用)
  └── ...
```

### 3.2 技术路线

#### 方案 A：按 SN 微调（推荐）

每条推进器独立微调一个专用模型。

```
训练：908 文件（SN01-SN12）→ 全局模型
微调：仅使用目标 SN 的 n 个文件 → 极小 lr 微调
推理：根据输入 CSV 的 SN → 选择对应模型
```

**优势：**
- 物理意义明确（每台推进器独立）
- v1 已验证有效

**代价：**
- 测试集 12 台推进器 → 12 个专用模型 → 存储 ~43MB
- 每个模型需额外微调 ~5-10 epoch（约 10-20min 每台）

#### 方案 B：按 test_mode + SN 分组微调

对误差较高的 mode（ssf、offmod、random_short）单独微调。

**优势：**
- 针对性强（ssf 单项 RMSE=0.1390N 是主要矛盾）
- 微调数据量更充足

#### 方案 C：Factory Calibration 模式（创新方案）

每台推进器出厂时用少量点火数据做快速微调（~1 epoch，~2min），生成个体修正向量。

```
全局预测 + 个体偏置修正 = 最终预测
```

---

### 3.3 微调细节

**模型改动：** 无需改架构，只改训练脚本。

```
冻结 attention 层，仅微调 FC head + LN scale：
  - FC(256→128) 的 weight/bias
  - FC(128→2) 的 weight/bias
  - LayerNorm 的 γ/β

学习率：
  - FC head: 1e-4（主要更新层）
  - LN scale: 1e-5（微调）

数据：
  - 目标 SN 的全部正常文件
  - 用 data_pipeline_v2._build_features() 读取
  - seq_len=200, stride=100
```

**损失函数：** 与全局训练一致：`0.7×MSE(thrust) + 0.3×MSE(mfr)`

### 3.4 预期效果

| 推进器 | 全局 RMSE | 微调后预期 |
|:-------|:---------:|:----------:|
| SN13（ssf） | 0.1390 N | ~0.09 N（-35%） |
| SN13（onmod） | 0.0212 N | ~0.015 N（-29%） |
| SN13（offmod） | 0.0967 N | ~0.07 N（-28%） |
| SN18（ssf 特差） | 推测 >0.15 N | ~0.10 N |

> 参考 v1 的 27.97% 改善幅度，预期 v2 可达 20-35% 改善。

## 四、App 页面整合

微调完成后，app_v2.py 按 SN 加载对应模型：

```python
# 根据上传 CSV 的 SN 选择模型
model_paths = {
    13: "outputs/models/v2/dual_output_lstm_v2_sn13.pth",
    14: "outputs/models/v2/dual_output_lstm_v2_sn14.pth",
    # ... 默认使用全局模型
}
sn = meta_row['sn']  # 从 metadata 获取
path = model_paths.get(sn, "outputs/models/v2/dual_output_lstm_v2.pth")
model = load_dual_model(path)
```

- 顶栏状态显示 `MODEL: GLOBAL` 或 `MODEL: SN13-TUNED`

## 六、风险

| 风险 | 可能性 | 应对 |
|:-----|:------:|:------|
| 某些 SN 数据量不足 | 低 | 数据增强（加噪声、时间抖动） |
| 微调过拟合（仅几十个文件） | 中 | 早停 patience=5，极小 lr |
| 微调后泛化性下降 | 中 | 保留全局模型作为 fallback |
| 对 test_mode 分项改善不一致 | 中 | 按 mode+SN 交叉分组 |
